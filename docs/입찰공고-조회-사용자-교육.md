# 입찰공고 조회 — 사용자 교육 자료

> **대상 화면:** 대시보드 → **입찰공고 조회** (`/dashboard/announcements`)  
> **작성 목적:** 수집된 KPOS 입찰공고를 조회·선별·후속 업무(발주보고·입찰·견적)로 연결하는 방법을 안내합니다.

---

## 1. 화면 개요

**입찰공고 조회**는 발주 기관(사이트)별로 수집된 공고를 한곳에서 확인하는 화면입니다.

| 항목 | 설명 |
|------|------|
| **주요 목적** | 공고 탐색, 관심 등록, 검토 선별, 발주·입찰·견적 업무로의 연결 |
| **데이터 출처** | KPOS 등 외부 사이트에서 수집된 공고 + 사용자가 직접 등록한 공고 |
| **기본 흐름** | 사이트 선택 → 공고 유형 선택 → (필터·검색) → 목록 확인 → 상세·작업 |

로그인 후 왼쪽 메뉴에서 **입찰공고 조회**를 선택하면 이 화면으로 이동합니다.

---

## 2. 화면 구성

화면은 위에서 아래로 다음 영역으로 구성됩니다.

```
┌─────────────────────────────────────────────────────────┐
│ ① 입찰공고 사이트 선택 (카드)                              │
├─────────────────────────────────────────────────────────┤
│ ② 검색 · 새로고침 · 공고 직접 등록 · 엑셀 다운로드           │
├─────────────────────────────────────────────────────────┤
│ ③ 필터 (관심공고 / 자동선별 / 공고일 / 마감 등)             │
├─────────────────────────────────────────────────────────┤
│ ④ 공고 유형 탭 (입찰공고 / 수의계약 / 발주계획·규격)        │
├─────────────────────────────────────────────────────────┤
│ ⑤ 공고 목록 테이블                                       │
├─────────────────────────────────────────────────────────┤
│ ⑥ 페이지 이동 (이전 / 다음)                               │
└─────────────────────────────────────────────────────────┘
```

목록 위에는 **현재 조건 요약**(선택 사이트, 공고 유형, 적용 필터, 총 건수)이 표시됩니다.

---

## 3. 공고 유형

상단 탭에서 공고 유형을 전환합니다. 유형마다 목록에 표시되는 열이 다릅니다.

| 탭 이름 | 설명 | 목록에 추가로 표시되는 정보 |
|---------|------|------------------------------|
| **입찰공고** | 일반 입찰 공고 | 상태, 구분, 입찰마감 |
| **수의계약 사전공고** | 수의계약 관련 사전 공고 | 주요구매내용 |
| **발주계획·규격 사전공고** | 발주 계획·규격 사전 공고 | (공통 열만 표시) |

모든 유형에 공통으로 **관심 · 선별 · 작업 · 공고번호 · 공고명 · 부서 · 공고일시 · 공고기간** 열이 있습니다.

---

## 4. 기본 사용 방법

### 4-1. 사이트 선택

1. 화면 상단 **입찰공고 사이트** 카드 중 조회할 기관을 클릭합니다.
2. 선택한 사이트는 브라우저에 기억되어, 다음 방문 시에도 유지됩니다.
3. 활성화된 사이트가 없으면 관리자에게 **관리자메뉴 → 입찰공고 조회 사이트 관리**에서 사이트 등록을 요청합니다.

### 4-2. 공고 검색

- 검색창에 **공고번호, 공고명, 부서** 중 일부를 입력하고 **검색**을 누릅니다.
- 현재 선택된 사이트·공고 유형·필터 조건 안에서만 검색됩니다.

### 4-3. 목록 새로고침

**새로고침** 버튼을 누르면 최신 수집 결과를 다시 불러옵니다.

### 4-4. 공고 상세 보기

- 목록에서 **공고명** 또는 해당 행을 클릭하면 **상세 창**이 열립니다.
- 상세 창에서는 공고 개요, 입찰 정보(입찰공고인 경우), 첨부파일, 개인 메모 등을 확인할 수 있습니다.

## 5. 필터 기능

여러 필터를 조합해 목록을 좁힐 수 있습니다.

| 필터 | 설명 |
|------|------|
| **관심공고만 보기** | 내가 ☆로 등록한 공고만 표시 |
| **자동선별** | 관리자가 등록한 키워드가 **공고명·공고번호·부서** 중 하나에 포함된 공고만 표시 |
| **공고일시 전일자만 보기** | 전일(또는 월요일에는 금·토·일) 공고만 표시 |
| **입찰마감 종료** | 마감이 지난 공고만 표시 (기본 목록은 진행 중 공고 위주) |

### 자동선별 (키워드 필터)

- **자동선별**을 켜면 관리자가 등록한 **활성 키워드**와 매칭되는 공고만 보입니다.
- 키워드 개수는 `(키워드 N개)` 형태로 표시됩니다.
- **키워드 보기** / **키워드 숨기기**로 등록된 키워드 목록을 펼치거나 접을 수 있습니다. (기본: 숨김)
- 키워드 등록·변경은 **관리자메뉴 → 자동선별 키워드**에서 합니다.

| 상황 | 화면 메시지 |
|------|-------------|
| 키워드가 하나도 없을 때 | 관리자 메뉴에서 키워드 등록 안내 |
| 키워드는 있으나 매칭 공고 없음 | 등록된 키워드에 해당하는 공고가 없습니다 |

### 대시보드에서 넘어온 필터

대시보드의 **마감 임박** 링크 등을 통해 들어오면 **7일 이내 마감**, **1일 이내 마감** 필터가 자동 적용될 수 있습니다.  
적용 중일 때는 노란색 안내와 **필터 해제** 링크가 표시됩니다.

## 6. 목록 열 기능

### 6-1. 관심 (☆)

| 동작 | 설명 |
|------|------|
| ☆ 클릭 | 관심 공고 등록 / 해제 |
| 등록 후 | **관심공고** 메뉴에서도 동일 공고 조회 가능 |
| 관심공고만 보기 | 필터 체크 시 관심 등록한 공고만 목록에 표시 |

### 6-2. 선별 (대기 / 미대상 / 대상)

개인별로 공고 검토 상태를 표시합니다. 버튼을 클릭할 때마다 순환합니다.

```
대기 → 미대상 → 대상 → 대기 …
```

| 상태 | 의미 | 목록 표시 |
|------|------|-----------|
| **대기** | 아직 검토 전 | 일반 표시 |
| **미대상** | 검토 후 제외 | 취소선·회색 |
| **대상** | 검토 후 관심/진행 대상 | 강조 표시 |

> 선별 상태는 **로그인한 사용자 본인**에게만 저장됩니다.

### 6-3. 작업 (발주보고 · 입찰하기 · 견적내기)

**작업 ▾** 메뉴에서 후속 업무를 시작합니다.

| 메뉴 | 최초 선택 시 | 이미 등록된 경우 |
|------|-------------|-----------------|
| **발주보고** | 발주보고 대상으로 등록 | **발주 화면으로** 이동 |
| **입찰하기** | 입찰 대상으로 등록 | **입찰 화면으로** 이동 |
| **견적내기** | 견적 대상으로 등록 | **견적 화면으로** 이동 |

등록이 완료되면 버튼 라벨이 **발주·입찰·견적** 등으로 바뀌어 진행 중임을 알 수 있습니다.

필터 영역 아래에는 **관심 등록 N건 · 입찰 등록 N건 · 견적 등록 N건** 요약도 표시됩니다.

## 7. 상세 창 기능

목록에서 공고를 클릭하면 열리는 상세 창에서 추가 작업이 가능합니다.

### 확인 가능한 정보

- **공고 개요:** 공고번호, 구분, 부서, 공고일시, 공고기간 등
- **입찰 정보:** (입찰공고) 진행상태, 입찰방식, 입찰개시·마감, 마감까지 남은 시간 등
- **수의계약 정보:** (수의계약 사전공고) 주요구매내용
- **첨부파일:** 등록된 첨부 목록 및 다운로드

### 상세 창에서 할 수 있는 작업

| 기능 | 설명 |
|------|------|
| **관심 등록/해제** | 목록과 동일 |
| **작업 메뉴** | 발주보고 · 입찰 · 견적 (목록과 동일) |
| **내 메모** | 공고별 개인 메모 작성·저장 (본인만 볼 수 있음) |
| **수정 / 삭제** | **직접 등록한 공고**에 한해 등록자가 수정·삭제 가능 |

상세 창 바깥을 클릭하거나 닫기로 창을 닫을 수 있습니다.

---

## 8. 공고 직접 등록

수집되지 않은 공고를 수동으로 등록할 때 사용합니다.

1. **공고 직접 등록** 버튼 클릭
2. 현재 선택된 **사이트·공고 유형**에 맞는 입력 양식 작성
3. 저장 후 목록에 반영되며, 상세 창에서 다시 확인 가능

직접 등록한 공고는 등록한 사용자가 **수정·삭제**할 수 있습니다.

---

## 9. 엑셀 다운로드

**엑셀 다운로드** 버튼으로 현재 조회 조건(사이트, 유형, 검색어, 필터)에 맞는 공고 목록을 파일로 저장합니다.

| 항목 | 내용 |
|------|------|
| 포함 정보 | 관심 여부, 공고번호, 공고구분, 공고명, 유형별 추가 열, 부서, 공고일시, 공고기간 |
| 제한 | 최대 10,000건까지 (그 이상은 잘림) |
| 파일명 | 사이트명·공고 유형·날짜가 포함된 `.xlsx` |

목록이 0건이면 다운로드 버튼이 비활성화됩니다.

---

## 10. 페이지 이동

- 목록은 **한 페이지당 20건**씩 표시됩니다.
- 하단 **이전 / 다음** 버튼과 `현재 페이지 / 전체 페이지`로 이동합니다.
- 필터·검색·유형을 바꾸면 1페이지부터 다시 조회됩니다.

## 12. 업무 시나리오 예시

### 시나리오 A — 매일 신규 공고 확인

1. 입찰공고 조회 접속
2. 사이트 선택 (예: 한수원)
3. **공고일시 전일자만 보기** 체크
4. **입찰공고** 탭에서 목록 확인
5. 관심 있는 공고에 ☆ 등록, 선별 상태를 **대상**으로 변경

### 시나리오 B — 키워드 기반 1차 필터링

1. **자동선별** 체크 (관리자가 등록한 키워드 적용)
2. 필요 시 **키워드 보기**로 적용 중인 키워드 확인
3. 남은 목록에서 상세 검토 후 **미대상** / **대상** 구분

### 시나리오 C — 마감 임박 공고 대응

1. 대시보드 **7일 이내 마감** 또는 **1일 이내 마감** 링크로 진입  
   *(또는 해당 조건으로 조회)*
2. 목록에서 **작업 → 입찰하기** 또는 **견적내기** 선택
3. 등록 후 해당 업무 메뉴에서 상세 진행

### 시나리오 D — 수집되지 않은 공고 등록

1. **공고 직접 등록**으로 공고 정보 입력
2. 목록·상세에서 확인
3. 이후 발주·입찰·견적 업무와 동일하게 **작업** 메뉴 사용

## 13. 자주 묻는 질문 (FAQ)

**Q. 목록이 비어 있습니다.**  
- 선택한 사이트·공고 유형에 해당 공고가 없거나, 필터가 너무 좁을 수 있습니다. 필터를 해제하고 **새로고침**해 보세요.

**Q. 자동선별을 켰는데 아무것도 안 나옵니다.**  
- 관리자가 키워드를 등록했는지 확인하세요. 키워드는 있으나 매칭 공고가 없을 수도 있습니다.

**Q. 관심을 해제했는데 관심공고만 보기 중이면 목록에서 사라집니다.**  
- 정상 동작입니다. **관심공고만 보기**를 해제하면 다시 표시됩니다.

**Q. 선별 상태는 동료와 공유되나요?**  
- 아닙니다. 선별·메모·관심은 **개인별**로 저장됩니다.

**Q. 수집 공고와 직접 등록 공고의 차이는?**  
- 수집 공고는 외부 사이트에서 자동으로 가져온 데이터이며, 직접 등록 공고는 사용자가 입력한 데이터입니다. 직접 등록 공고만 등록자가 수정·삭제할 수 있습니다.

**Q. 엑셀과 화면 건수가 다를 수 있나요?**  
- 동일 조건 기준이지만, 엑셀은 최대 10,000건까지만 포함합니다.
